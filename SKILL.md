# Seedance KOR Storyforge

## 트리거 문구
- "영상 프롬프트 만들어줘"
- "Seedance/즉몽 프롬프트"
- "15초 넘는 영상 분할"
- "광고/숏드라마/액션 콘티"

## 출력 원칙
- 한국어로 작성한다.
- 질문은 최대 2개까지만 한다.
- 결과를 먼저 주고 설명은 최소화한다.
- 항상 아래 3가지를 포함한다.
  1) 메인 결과물
  2) 안정화 버전(복잡도 축소)
  3) Self-check 결과

## 플랫폼 하드 리밋 (Seedance 2.0)
- 이미지: 최대 9장 (`@이미지1~9`), 각 30MB 이하
- 영상: 최대 3개 (`@영상1~3`), 총 2~15초, 각 50MB 이하
- 오디오: 최대 3개 (`@오디오1~3`), 총 15초 이하, 각 15MB 이하
- 총 첨부: 최대 12개
- 단일 생성 길이: 4~15초

리밋 초과가 예상되면 즉시 축소안을 제시한다.

## 기본 실행 순서
1. 사용자 의도 분류(광고/드라마/액션/정보/연재)
2. 출력 모드 선택(director-mode 또는 executor-mode)
3. 길이 판단(≤15초 단일 / >15초 분할-연장)
4. 참조 자산 역할 라벨 부여
5. A/B/C 버전 출력 + 안정화 버전 + 재시도안
6. Self-check 5항목 출력

## 출력 모드
### 1) director-mode
- 사용자 역할: 크리에이티브 디렉터(지시/승인)
- 출력: 목표/제약/승인기준 + 제작안

### 2) executor-mode
- 사용자 역할: 즉시 제작
- 출력: 복붙 가능한 프롬프트 중심

세부 구조는 `references/output-modes-ko.md` 참고.

## 길이별 출력
### A) 15초 이하
아래 순서로 출력한다.
1. 프로젝트 요약
2. A/B/C 버전 프롬프트
3. 안정화 버전(20~30% 단순화)
4. 실패 시 재시도안(복잡도 30% 축소)
5. Self-check

### B) 15초 초과
- SEG1: 신규 생성
- SEG2+: `@영상1 연장`
- 각 SEG에 "엔드프레임 메모" 필수
- 다음 SEG 첫 줄에 이전 엔드프레임 재선언 필수

## 참조 자산 역할 라벨 규칙
출력 시 반드시 아래처럼 명시한다.
- `@이미지1: 캐릭터/제품 기준`
- `@이미지2: 배경/무드`
- `@영상1: 카메라/액션 레퍼런스`
- `@오디오1: BGM/톤`

역할 라벨이 불명확하면 품질 저하 위험을 경고한다.

## 카메라 레벨 시스템
- **Basic**: 1개 카메라 동작(안정형)
- **Pro**: 2개 조합(표준)
- **Epic**: 2~3개 + 질감 표현(실험형)

과도한 조합은 금지한다(최대 3개).

## 모드 선택 규칙
- 광고/제품: 재질/광택/회전/클로즈업
- 드라마: 감정선/표정/대사 타이밍
- 액션/판타지: 동작 선명도/속도 제어
- 정보/교육: 단계적 시각화/과장 최소화
- 연재형: 에피소드 엔드프레임 연결
- multi-agent-plan: 이미지/영상/카피 병렬 분할 제안

## Fallback 규칙
실패 가능성이 있으면 아래 문구를 포함한다.
- "1차 생성 실패 시: 전환 수와 카메라 조합을 30% 축소해 재시도"
- "참조 자산 우선순위: 캐릭터/제품 > 카메라 > 오디오"

## Self-check (항상 출력)
아래 5개를 체크한다.
- [ ] @참조 번호와 역할 라벨이 매칭되는가
- [ ] 총 파일 수/길이/용량 제한을 만족하는가
- [ ] 타임라인 구간이 전체 시간을 커버하는가
- [ ] 자막/워터마크/텍스트 오버레이 금지 조건이 있는가
- [ ] 안정화 버전과 재시도안이 포함됐는가

상세 체크 포맷은 `references/self-check-ko.md` 참고.
