# Seedance KOR Storyforge

## 트리거 문구

다음 의도가 보이면 바로 이 스킬을 적용한다.

- "영상 프롬프트 만들어줘"
- "Seedance 프롬프트"
- "즉몽/Seedance2로 숏폼 만들고 싶어"
- "광고 영상 콘티 짜줘"
- "15초 넘는 영상 분할해서 만들어줘"

## 출력 원칙

- 한국어로 작성한다.
- 설명보다 결과물을 우선한다.
- 먼저 바로 쓸 수 있는 **메인 프롬프트 1안**을 제공한다.
- 필요 시 **대안 1~2안**을 추가한다.

## 기본 실행 순서

1. 사용자 의도에서 장르/목표를 즉시 분류한다.
2. 입력이 부족해도 가정값으로 초안을 먼저 생성한다.
3. 길이에 따라 출력 포맷을 선택한다.
   - 15초 이하: 단일 타임라인 프롬프트
   - 15초 초과: 분할 + 연장(@영상1) 플랜
4. 참조 파일이 있으면 `@이미지N`, `@영상N`, `@오디오N` 문법으로 반영한다.
5. 마지막에 금지/회피 조건(워터마크, 자막, 깨짐 등)을 붙인다.

## 플랫폼 하드 리밋 (Seedance 2.0)

- 이미지: 최대 9장 (`@이미지1~9`), 각 파일 30MB 이하
- 영상: 최대 3개 (`@영상1~3`), 총 길이 2~15초, 각 파일 50MB 이하
- 오디오: 최대 3개 (`@오디오1~3`), 총 길이 15초 이하, 각 파일 15MB 이하
- 총 첨부 수: 최대 12개 (이미지+영상+오디오)
- 단일 생성 길이: 4~15초

리밋 초과가 예상되면 먼저 축소안(참조 개수/길이/복잡도 절감)을 제시한다.

## 기본 가정값 (정보 없을 때)

- 길이: 15초
- 비율: 9:16 (숏폼 기본)
- 스타일: 시네마틱 + 자연광 + 과한 이펙트 억제
- 카메라: 3구간(도입-전개-클로징)
- 사운드: 배경음 + 핵심 효과음 + 대사 최대 1문장

## 비율/목적 프리셋

- **9:16 (숏폼/릴스/틱톡)**: 인물 중심 중근경 비중↑, 세로 동선 강조, 중앙 안전영역 유지
- **16:9 (유튜브/시네마틱)**: 배경 정보량↑, 와이드 establishing shot 1회 포함
- **1:1 (피드 광고)**: 중앙 구도 고정, 좌우 과한 이동 최소화

정보가 없으면 9:16을 기본 적용한다.

## 출력 포맷

### A) 15초 이하 단일 포맷

```text
[프로젝트 요약]
- 목표:
- 톤/장르:
- 길이/비율:

[Seedance Prompt]
0-3초: ...
4-9초: ...
10-15초: ...

[사운드]
- BGM:
- SFX:
- 대사/내레이션:

[참조]
- @이미지1: ...
- @영상1: ...
- @오디오1: ...

[네거티브/제한]
- 워터마크/자막/텍스트 오버레이 없음
- 얼굴/손 디테일 붕괴 방지
- 과도한 깜빡임/왜곡/노이즈 억제

[안정화 버전]
- 위 프롬프트의 이벤트 수를 20~30% 줄인 간소화 버전 1개를 추가 제시
```

### B) 15초 초과 분할-연장 포맷

```text
[총괄]
- 총 길이: 30초
- 구성: 15초 x 2세그먼트

[SEG1 | 0-15초 | 신규 생성]
...타임라인 프롬프트...
- 엔드프레임 메모: (다음 세그 연결용)

[SEG2 | 15-30초 | @영상1 연장]
- 지시: "@영상1을 15초 연장"
...타임라인 프롬프트...

[연결 안정화 규칙]
- 주인공 의상/헤어/색상 고정
- 배경 시간대/광원 연속성 유지
- 카메라 진행방향 반전 금지
```

## 모드 선택 규칙

- 광고/제품: 제품 디테일 + 재질 + 회전/클로즈업 강조
- 드라마/대사: 감정선 + 표정 + 대사 타이밍 우선
- 액션/판타지: 동작 선명도 + 카메라 속도 제어 + 충돌 사운드
- 정보/교육: 단계적 시각화 + 과장 연출 최소화
- 연재형: 에피소드별 엔드프레임 메모 필수

세부 템플릿은 `references/templates-ko.md`를 읽어 적용한다.
즉시 사용 가능한 샘플은 `references/examples-ko.md`를 우선 참조한다.

## 실패 방지 규칙

- 한 프롬프트에 이벤트를 과도하게 넣지 않는다.
- 장면 전환은 2~3회 이내로 제한한다.
- 대사는 짧고 명확하게 유지한다.
- 민감/차단 가능 표현은 우회 표현으로 정리한다.
- 항상 메인 버전과 함께 묘사 밀도를 낮춘 "안정화 버전"을 제시한다.

실패 패턴과 교정법은 `references/troubleshooting-ko.md`를 우선 참조한다.

## 응답 스타일

- 질문 폭탄 금지. 필요한 확인은 최대 2개.
- 사용자가 "바로"를 원하면 확인 질문 없이 초안부터 제공.
- 최종 응답은 복붙 가능한 텍스트 중심으로 작성한다.
